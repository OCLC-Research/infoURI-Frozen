description 
The ofi namespace was introduced by the NISO AX Standardization Committee to
facilitate the operation of the Registry defined in the NISO Z39.88-2004
Standard "The OpenURL Framework for Context-Sensitive Services".  Entries in
that OpenURL Registry are assigned a Registry Identifier, which is of one of
the following forms (hereby _ stands for a string consisting of alphabetic
characters, numeric characters, the period and the colon):
	info:ofi/nam:_ for Registry Identifiers of Namespaces used in
        implementations of the OpenURL Framework Standard
	info:ofi/enc:_ for Registry Identifiers of Character Encodings used in
        implementations of the OpenURL Framework Standard
	info:ofi/fmt:_  for Registry Identifiers of Physical Representations
        used in implementations of the OpenURL Framework Standard
	info:ofi/fmt:_:_  for Registry Identifiers of Constraint Languages
        used in implementations of the OpenURL Framework Standard
	info:ofi/fmt:_:_:ctx   for Registry Identifiers of ContextObject
        Formats used in implementations of the OpenURL Framework Standard (ctx
        is a mandatory prefix)
	info:ofi/fmt:_:_:_   for Registry Identifiers of Metadata Formats used
        in implementations of the OpenURL Framework Standard (ctx must not be
        used as a prefix for the last part of this Registry Identifier)
	info:ofi/tsp:_   for Registry Identifiers of Transports used in
        implementations of the OpenURL Framework Standard
	info:ofi/pro:_   for Registry Identifiers of Profiles used in
        implementations of the OpenURL Framework Standard

Examples:
	info:ofi/enc:UTF-8
	info:ofi/fmt:kev
	info:ofi/fmt:kev:mtx
	info:ofi/fmt:xml:xsd:xsd
	info:ofi/fmt:kev:mtx:book
	info:ofi/tsp:https:openurl-by-ref
	info:ofi/pro:sap-1
