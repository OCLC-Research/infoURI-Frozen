identifier-syntax
description 
There are a number of SRW object types that require unambiguous identifiers in
order for the protocol to function properly.  These include CQL context sets
and record schemas.  The syntax would thus be:
info:srw/<srw object type>/<sub-authority>/<id>

The <id> is opaque as far as this concerns the info syntax, but may
itself be structured later. 

example:
info:srw/cql-context-set/1/cql-v1.1
uri http://www.loc.gov/sru/infoURI.html
