identifier-syntax
description 
As an "info" URI an ARK has the form, NAAN + "/" + Name, where NAAN is a globally unique Name Assigning Authority Number given to an organization that assigns ARKs and Name is a string assigned by that organization.  A NAAN is a 5- or 9-digit number and a Name is a string of ASCII letters, digits, and the characters { = # * + @ _ $ }, not including the braces.  In addition, a Name may contain the characters { . / % - }, which are reserved to indicate variant form, logical containment, escaped octet, and a lexically ignorable separator, respectively.
uri http://www.cdlib.org/inside/diglib/ark/arkspec.pdf
identifier-normalization
description 
ARKs are designed to be embeddable in URLs where the protocol and hostname provide branded (the current provider) actionability but are formally identity-inert in the sense that they do not participate in comparing two ARKs for equality.  To render an ARK as an "info" URI, first remove the protocol and hostname, if any; then remove any hyphens and any initial or final occurrences of { . / }.  The resulting string is case sensitive.
