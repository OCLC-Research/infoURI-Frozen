identifier-syntax
description 
The syntax of the handle string is: PREFIX"/"SUFFIX where PREFIX is a globally unique string identifying the administrative unit of the handle; and SUFFIX is any unique "local name" chosen by the prefix holder. Handles may consist of any printable characters from the UCS-2 character set. There is no limit on the length of a handle string, nor on any of its components. A handle SUFFIX may contain any characters defined in UCS-2. A handle PREFIX may contain any characters defined in UCS-2, except the character '/'.
uri http://www.ietf.org/rfc/rfc3650.txt
identifier-normalization
description 
By default, handles are case sensitive; however, any handle service may define its namespace under which all ASCII characters within any identifier are case insensitive.
